Bipartite Matching

Example Bipartite Matching

Given is an undirected graph G = (V,E). The set of vertices is divided into two sets A and B which have no vertex in common. Each edge connects a vertex of A with one in B. A matching M is a subset of E such that every two elements of M have no common edge. The problem is the determination of a so-called maximal matching, i.e. a matching of maximal cardinality. For more information see Bipartite graph - Wikipedia [last called 2023-09-21].

We apply the method of augmenting paths. The algorithm and more interesting facts can be found in Levin, Oscar: Discrete Mathematics: An Open Introduction, 3rd edition [last called 2023-09-21]. A path is a finite ordered sequence of vertices. Here only paths are considered whose edges are edges of G. That means that vertices of A and B alternate with each other. The green and red edges mark a path (b,f,a,h,d,j) or (j,d,h,a,f,b). Let us choose the first orientation. Edges that belong to a matching are colored green, the rest is colored red. When we walk along the path we recognize that {b,f}, {a,h} and {d,j} belong to a matching M.



Now consider a path that begins at a vertex of A that does not belong to M, for example e. We can augment the path by adding the edge {e,j} and also {b,g}. So we get a new path (e,j,d,h,a,f,b,g). If we walk along the path we recognize that {e,j}, {d,h}, {a,f} and {b,g} belong to a new Matching M1. These edges are now colored green. But the edges of the former matching M do not belong to the new matching any more. They are colored red. By this process we got a new matching M1 consisting of one element more than M.



This procedure can be continued. A new augmenting path is (c,g,b,f,a,h,d,j,e,k). This results in a perfect matching M2 = {{c,g}, {b,f}, {a,h}, {d,j}, {e,k}}. It is called perfect because all vertices of A and B belong to an edge of M2.



The code written in Python 3.9 contains three recursive functions needed to find augmenting paths and to build larger matchings if possible. You can easily modify the code inserting your own sets A, B, each consisting of at most five vertices, and a predefined matching and your set of arcs. For the program the edges are initially directed from vertices of A to vertices of B, so they are called arcs instead of edges. Your initial matching may be empty, M = []. It may happen that a new path is very short, consisting only of two vertices. A new matching may consist of more than one path.
